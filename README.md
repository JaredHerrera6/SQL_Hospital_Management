## Operations

This Project analyzes hospital operational data using SQL to uncover insights related to patient visits, treatment demand, and revenue generaltion.

The objective is to simulate the work of a data analyst in a healthcare organization, where SQL is used to evaluate operational performance and support data-driven decision making. 

# Business Problem
Hospitals must monitor patient flow, treatment demand, and revenue streams to improve staffing, scheduling, and financial performance.

This Project answers several key questions:

- Which doctors handkle the most appointments?
- What treatments are performed most frequesntly?
- Which patients visit the hospital most often?
- What treatments generate the most revenue?
- What payment methods contribute the most to hopsital revenue?

 # SQL Analysis
 ## Doctor Workload
 ```` sql
 SELECT 
    d.first_name,
    d.last_name,
    COUNT(a.appointment_id) AS total_appointments
FROM doctors d
JOIN appointments a
ON d.doctor_id = a.doctor_id
GROUP BY d.doctor_id, d.first_name, d.last_name
ORDER BY total_appointments DESC;
 ````
Insight:

Some doctors manage significantly more appointments than others, suggesting potential scheduling imbalance.

## Most Common Treatments
```` sql
SELECT 
    treatment_type,
    COUNT(*) AS treatment_count
FROM treatments
GROUP BY treatment_type
ORDER BY treatment_count DESC; 
````
Insight:

MRI procedures appear most frequently in the dataset.

## Patients with multiple Visits
````sql
SELECT 
    p.first_name,
    p.last_name,
    COUNT(a.appointment_id) AS visits
FROM patients p
JOIN appointments a
ON p.patient_id = a.patient_id
GROUP BY p.patient_id, p.first_name, p.last_name
HAVING COUNT(a.appointment_id) > 1;
````
Insight:

All but two patients visit the hospital multiple times, indication ongoing care needs. 

## Revenue by Payment Method
````sql
SELECT 
    payment_method,
    SUM(amount) AS revenue
FROM bills
GROUP BY payment_method;
````

Insight:
Credit Card payments generate the majority of hospital revenue.

## Revenue Generated by Each Doctor
```` sql
SELECT
    d.first_name,
    d.last_name,
    SUM(t.cost) AS doctor_revenue
FROM doctors d
JOIN appointments a
ON d.doctor_id = a.doctor_id
JOIN treatments t
ON a.appointment_id = t.appointment_id
GROUP BY d.doctor_id, d.first_name, d.last_name
ORDER BY doctor_revenue DESC;
````

## Average treament Cost by Treatment Type
````sql
SELECT
    treatment_id,
    treatment_type,
    cost,
    AVG(cost) OVER (PARTITION BY treatment_type) AS avg_cost
FROM treatments;
````
This window functiontion allows comparison of each treatments cost against the average cost of its category.

## Most Recent appointment for each patient
````sql
SELECT
    p.first_name,
    p.last_name,
    a.appointment_date
FROM (
    SELECT *,
           ROW_NUMBER() OVER (
               PARTITION BY patient_id
               ORDER BY appointment_date DESC
           ) AS rn
    FROM appointments
) a
JOIN patients p
ON p.patient_id = a.patient_id
WHERE rn = 1;
````
Shows the most recent appointment for each patient allowing the hospital to see if a patient is coming back too soon or too delayed.

# Key Insights

- Credit Card payments dominate this hospital's revenue streams.
- Some Doctors handle siginificantly higher appointment volumes.
